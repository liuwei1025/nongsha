# 农沙 (Nongsha)

农沙是一个本地优先的 PWA，适用于你有一小段时间、头脑混乱、不知道下一步该做什么的时刻。

应用将原始笔记保存在设备本地，只在你明确要求时才发送到轻量级解析端点，然后将结果转换为可编辑的目标和任务。推荐结果本地存储且确定性生成，让应用能够解释自己的行为，而不是像个黑盒子的老虎机。

## 功能现状

- `捕获` 页面：原始笔记输入和显式解析
- 可编辑的结构化草稿，支持字段锁定，用户编辑优先于后续重新解析
- `首页`：本地推荐评分、轻量级周/日标记、执行日志
- `报告` 页面：可信的进度指标，而非虚假的精确计时
- `/api/parse` 轻量级 Express API
- 通过 Dexie 实现 IndexedDB 持久化
- 合并、评分、报告和解析器契约的单元测试
- 核心捕获 → 推荐 → 完成 → 报告流程的 Playwright 端到端测试

## 技术栈

- React 19 + TypeScript
- Vite 8
- Express 5（轻量级解析 API 和统一开发服务器）
- Dexie（本地优先 IndexedDB 存储）
- Vitest + Playwright（测试覆盖）

## 解析模式

农沙支持两种解析模式：

1. `远程解析`
   如果配置了兼容的 OpenAI 风格 provider，解析端点会优先尝试远程模型。
2. `启发式后备`
   如果未配置 provider 或请求失败，农沙会回退到确定性本地解析器，保证产品仍能正常工作。

服务器无需持久化原始笔记进行解析。

## 环境变量

如需远程解析，请复制 `.env.example`。

目前支持的变量：

- `OPENAI_API_KEY`
- `OPENAI_BASE_URL`
- `OPENAI_MODEL`
- `ARK_API_KEY`
- `ARK_BASE_URL`
- `ARK_MODEL`
- `DISABLE_REMOTE_PARSE=true`

## 代码中的重要产品规则

- 原始笔记是捕获和可审计性的规范来源。
- 用户编辑的结构化字段在重新解析时保持权威性。
- 推荐结果在本地派生，但每个显示的推荐都会获得持久化的快照事件。
- 执行创建自己的事件，使报告和历史记录在后续评分变化时不会漂移。

## 多语言

农沙支持多语言界面，包括英语和简体中文。

应用会自动检测浏览器语言设置，也可以在设置中手动切换。

如需添加新的语言支持：

1. 在 `src/i18n/locales/` 目录下创建新的语言文件（如 `ja.json`）
2. 在 `src/i18n/index.ts` 中的 `languages` 数组和 `resources` 对象中添加新语言
3. 翻译所有相关文本
